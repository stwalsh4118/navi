# [16-10] Add PR link action in git detail view

[Back to task list](./tasks.md)

## Description

Allow users to open GitHub PR or issue links directly from the git detail view. When a PR/issue is detected from the branch name, pressing Enter or `o` opens the URL in the system default browser.

## Requirements

- Show PR/issue link in git detail view when detected
- `Enter` or `o` opens the link in default browser
- Use `xdg-open` (Linux), `open` (macOS), or `start` (Windows)
- Show "No PR/issue detected" if no link available
- Construct URL from GitHub info: `https://github.com/owner/repo/issues/123`
- Display visual indicator that link is actionable

## Implementation Plan

1. Implement `openURL(url string) error` using platform-specific command
2. Construct PR/issue URL from GitHubInfo and detected number
3. Add `Enter`/`o` keybinding in git detail view
4. Show PR link prominently in git detail view
5. Provide feedback if link opened successfully
6. Handle case when no link is available
