# [19-5] Update view to display remote sessions

## Description

Update the TUI view to visually distinguish remote sessions from local sessions. This includes showing the remote name label, displaying connection status indicators, and handling the display of offline/unavailable remotes.

## Requirements

- Display remote name in brackets after session name: `session-name [dev-server]`
- Show connection status indicator in header or footer (per remote)
- Visual distinction for remote sessions (subtle styling difference)
- Handle offline remotes: show cached sessions as stale or hide them
- Show "freshness" indicator for remote data (time since last successful poll)
- Update footer help text if remote-specific actions are available

## Implementation Plan

1. Update `renderSession()` in `view.go`:
   - Add remote label after session name when `s.Remote != ""`
   - Use dimmed style for remote label
2. Add remote status to Model:
   - Map of remote name to status (connected/disconnected/last poll time)
3. Update `renderHeader()` or add status line:
   - Show connection indicators for configured remotes
   - Format: `[dev-server:✓] [staging:✗]` or similar
4. Add styling constants for remote-related elements:
   - Remote label style
   - Status indicator colors (green for connected, red for disconnected)
5. Update `renderFooter()` to show remote count if remotes configured
